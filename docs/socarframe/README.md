# socarframe — SOCAR design system (style guide)

The knowledge base every harness stage reads when designing a SOCAR app screen. It is a
faithful transcription of **SOCARFRAME 2.0** (https://socarframe.socar.kr), regenerated by
`tools/socarframe_scrape.py`. Use tokens and components **by name** — never invent hex/px.

## The non-negotiables (apply on every screen)

- **Semantic tokens for UI.** Use `primary-regular` for the main CTA, `text-strong/primary/
  secondary/tertiary/disabled` for text hierarchy, `status-*` for feedback, `background-regular`
  for surfaces, `border/divider-*` for separators. Reserve raw palette swatches for illustration.
- **One primary action per screen**, emphasized with `primary-regular`; everything else supports it.
- **Type by token:** screen/section titles use `heading*`/`title*`, body uses `body*`, captions
  use `caption*`. Buttons use the size's type token (e.g. `title2` for a large ActionButton).
- **Spacing on the 4px grid:** typical screen padding `spacing-400` (16px); section gaps
  `spacing-300`/`spacing-400`; tight groups `spacing-100`/`spacing-200`.
- **Radius by token:** surfaces/cards `radius-300` (12px); buttons per size (`large` = `radius-350`
  = 14px). Match the px to the token name — `radius-350` is 14px, not 12px.
- **Font:** build in Figma with **Pretendard** (full weight set) unless the real SOCAR font is
  installed; never drop Semi Bold (600) to Medium (500).
- **Reuse components** from this guide before hand-building; build custom only for true gaps.
