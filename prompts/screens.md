# 디자인 대상 화면 & 초기 프롬프트 모음

`/auto` 하니스로 쏘카 앱 화면을 디자인할 때 입력하는 **초기 프롬프트** 모음이다.
`/auto 쏘카 앱의 쿠폰함 디자인해줘` 같은 한 줄짜리가 아니라, **실제 디자이너가 브리프를 적듯**
맥락 · 구성 · 상태 · 톤을 적당히 담아둔 형태로 작성했다. 그대로 복사해서 `/auto` 뒤에 붙여 쓰면 된다.

```
/auto <아래 화면별 프롬프트 블록 전체>
```

## 공통 전제 (모든 프롬프트에 기본 적용 — 프롬프트마다 반복하지 않음)

- **디바이스**: 모바일 포트레이트, 전체 화면(360 width 기준).
- **디자인 시스템**: socarframe 시맨틱 토큰/컴포넌트를 이름으로 사용. raw hex/px 지양.
- **폰트**: Pretendard 풀 웨이트.
- **보이스**: 한국어, 쏘카 톤(명료·따뜻·간결).
- **Figma 타겟**: 기본 파일(`VuBHVaAnA5ORacxMZ9zcH8`). 다른 파일을 쓰려면 프롬프트에 Figma URL을 덧붙인다.
- 각 프롬프트는 디자이너 브리프 수준의 "적당한 구체성"이다. 세부는 plan/implement 단계가 socarframe 기준으로 채운다.

## 구체성 표기

- **상**: 거의 다 명세됨 → clarify가 1라운드에서 READY 날 가능성이 높음(헤드리스에도 안전).
- **중**: 합리적 가정으로 진행하거나 1~2개만 되물음.
- **하**: 의도적으로 열어둠 → clarify 되묻기 루프를 태우기 좋음(인터랙티브 권장).

## 1. 홈 (메인 대시보드)  · 구체성: 중

```
쏘카 앱의 '홈(메인)' 화면을 디자인해줘. 앱을 켜면 처음 보는 화면으로, 예약 진입과 현재/예정 이용을 한눈에 보여주는 게 목표야.

구성:
- 상단 바: 현재 위치(또는 자주 가는 목적지) 주소를 보여주고, 우측에 알림 아이콘(읽지 않은 알림이 있으면 dot badge).
- 검색 진입: "어디서 탈까요?" 같은 큰 검색 박스. 누르면 쏘카존/목적지 검색으로 가는 진입점이야.
- 진행 중/예정 예약 카드: 차량 썸네일 + 모델명 + 이용 시간 + 쏘카존, 그리고 바로 '스마트키' 또는 '예약 상세'로 가는 버튼.
- 추천/혜택 배너 carousel: 도트 인디케이터 있는 2~3장(신규 할인, 주행 리워드 등).
- 퀵메뉴: 아이콘 + 라벨 그리드(쏘카플랜, 부름/편도, 쿠폰함, 이벤트 등).

상태: 예약이 있는 버전과 예약이 없는(첫 화면) 버전 두 가지로 보여줘.
톤: 활기차고 깔끔하게, 다음 행동(예약)을 자연스럽게 유도.
```

_왜: 컴포넌트 밀도가 가장 높은 플래그십 화면. Carousel·Badge·카드·퀵메뉴 그리드를 한 화면에서 보여준다. 빈/채워진 두 상태도 exercise._

## 2. 쏘카존 탐색 (지도 + 차량 리스트 바텀시트)  · 구체성: 중

```
쏘카 앱의 '쏘카존 탐색' 화면을 디자인해줘. 검색 후 지도에서 주변 쏘카존과 차량을 골라보는 화면이야.

구성:
- 배경: 풀스크린 지도. 쏘카존 핀 마커들이 찍혀 있고 선택된 핀은 강조. (지도 자체는 플레이스홀더로 둬도 돼)
- 상단: 뒤로가기 + 현재 검색어와 이용 날짜·시간을 요약해 보여주는 바.
- 필터: 차종(경차/SUV/전기차), 가격, 즉시예약 같은 Chip을 가로 스크롤로.
- 하단 바텀시트(중간 높이로 열려 있는 상태): "내 주변 OO대" 헤더 아래 차량 리스트 카드들 — 모델명, 쏘카존 이름·도보 거리, 시간당 요금, 상태 태그. 카드 누르면 상세로.

상태: 기본(차량 리스트가 있는 상태). 여유 되면 '주변에 차량 없음' 빈 상태도.
톤: 정보가 많아도 정돈되게, 차를 빠르게 고르도록.
```

_왜: BottomSheet(detent) + Chip 필터 + 지도 오버레이 + 리스트 카드라는, 홈과 전혀 다른 레이아웃 패턴을 검증. 빈 상태 옵션으로 강건성도 본다._

---

## 3. 차량 상세 & 옵션 선택  · 구체성: 상

```
쏘카 앱의 '차량 상세 & 옵션 선택' 화면을 디자인해줘. 차량 한 대를 고른 뒤 보험·옵션을 정하고 예약으로 넘어가기 직전 화면이야.

구성:
- 상단: 차량 이미지 carousel(여러 컷, 도트 인디케이터). 좌측 뒤로가기, 우측 공유/찜 아이콘.
- 차량 정보: 모델명, 번호판·연식, 연료·인승 태그, 쏘카존 위치 한 줄.
- 차량손해면책(보험) 선택: 표준 / 안심 / 완전 3단계를 선택형 박스(또는 라디오)로. 각 항목에 자기부담금과 추가 요금 표기, 선택된 항목 강조.
- 추가 옵션: 하이패스, 충전 등 체크/토글.
- 요금 안내: 대여요금 + 선택한 보험료가 반영된 예상 요금 미리보기.
- 하단 고정 영역: '예약하기'(primary, 가장 강조)와 그 좌측에 총 예상 요금.

상태: 기본(보험은 '안심'이 선택된 상태)만 디자인하면 돼.
톤: 신뢰감 있게, 선택을 어렵지 않게.
```

_왜: SelectionBox/Radio 선택 위계, Tag, 이미지 Carousel, 하단 고정 CTA + 가격 묶음. 잘 명세되어 clarify가 바로 READY 날 가능성이 높아 헤드리스에도 안전._

---

## 4. 예약·결제 확인  · 구체성: 상

```
쏘카 앱의 '예약·결제 확인' 화면을 디자인해줘. 예약 직전 마지막으로 내용을 확인하고 결제하는 전체 화면이야.

구성:
- 상단 바: '예약 확인'.
- 예약 요약 카드: 차량 / 쏘카존 / 이용 일시 / 대여 시간.
- 요금 상세: 대여요금, 보험료, 예상 주행요금 항목과 합계(강조). 세부는 아코디언으로 펼쳐볼 수 있게.
- 할인 적용: 쿠폰 선택(적용된 쿠폰 또는 '쿠폰 적용하기')과 포인트 사용 입력.
- 결제수단: 등록된 기본 카드 표시 + '변경'.
- 약관 동의: 필수/선택을 구분한 체크.
- 하단 고정 영역: 최종 결제 금액 + '결제하고 예약 완료'(primary).

상태: 기본 상태 하나만.
톤: 명료하고 안심되게. 금액이 또렷하게 읽히도록.
```

_왜: 폼 밀도가 높은 화면 — Accordion, Checkbox(필수/선택), 금액 강조 위계, 하단 결제 CTA. 결제는 신뢰가 핵심이라 토큰 위계 검증에 좋다._

## 5. 이용 중 (스마트키 제어)  · 구체성: 중

```
쏘카 앱의 '이용 중(스마트키)' 화면을 디자인해줘. 차량을 빌려 이용하는 동안 차 문을 제어하고 시간을 관리하는 화면이야.

구성:
- 상단: 차량 모델 · 번호판, 남은 이용 시간을 크게(반납 예정 시각도 함께).
- 핵심 컨트롤: 문 열기 / 잠그기, 비상등, 경적(차 찾기)을 큰 버튼으로. 지금 잠겨있는지 상태가 보이게.
- 안내 Tips: "탑승 전 차량 외관을 확인해 주세요" 같은 보조 안내.
- 주요 액션: 이용 시간 연장, 반납하기.
- 하단/구석: 사고·고장 신고와 고객센터로 가는 도움 진입.

상태: 기본은 '잠김' 상태. 가능하면 '열림' 상태 버전도.
톤: 안심되고 즉각적이게. 손이 바쁠 수 있으니 터치 타깃을 크게.
```

_왜: 버튼 중심의 컨트롤 화면 — IconButton/ActionButton, Tips, 큰 터치 타깃, 상태 토글. 정보형이 아닌 '조작형' 레이아웃을 검증._

---

## 6. 이용 내역 & 영수증  · 구체성: 중

```
쏘카 앱의 '이용 내역 & 영수증' 화면을 디자인해줘. 지난 이용을 리스트로 보고, 한 건을 눌러 영수증을 자세히 보는 흐름이야.

구성:
- 리스트: 월별로 구분된 이용 내역 카드 — 차량 모델, 이용 일시, 쏘카존, 결제 금액. 누르면 영수증 상세로.
- 영수증 상세: 이용 요약(차량·쏘카존·이용 일시·주행 거리) + 결제 영수증(대여/주행/보험/할인 항목별 금액과 총 결제 금액 강조, 세부는 아코디언). 결제수단과 승인 일시.
- 보조 액션: 영수증 문의, 같은 차 다시 예약(텍스트/보조 버튼).

상태: 리스트 + 영수증 상세 두 화면. 내역이 하나도 없을 때 빈 상태도 함께.
톤: 정확하고 신뢰감 있게. 숫자 가독성이 제일 중요.
```

_왜: 리스트 → 상세 두 화면 + 빈 상태. Accordion 금액 breakdown, 숫자 정렬/강조, 월별 구분. 한 번에 여러 프레임을 만드는 케이스._

## 7. 운전면허 등록 (온보딩)  · 구체성: 중하

```
쏘카 앱의 '운전면허 등록' 화면을 디자인해줘. 첫 예약 전에 운전면허를 등록·인증하는 온보딩 단계야.

구성:
- 상단: "운전면허를 등록해 주세요" 타이틀 + 한 줄 설명.
- 입력 폼: 면허 종류, 면허번호, 발급일자(날짜 선택), 이름·생년월일 등.
- 면허증 촬영: 카메라로 찍거나 업로드하는 영역과 촬영 안내 Tips.
- 동의: 본인확인/약관 체크.
- 하단 고정 영역: '등록하기'(primary).

상태: 입력 기본 상태와 인증 완료 상태를 같이 보여줘. (검증 중 로딩이 있으면 더 좋고)
톤: 신뢰감 있고 간결하게. 막히지 않고 끝까지 안내되도록.
```

_왜: Input/DatePicker/Checkbox 등 폼 컴포넌트 + 촬영 안내 + 상태(기본/완료). '상태를 같이'가 다소 열려 있어 clarify가 범위를 한 번 되물을 수 있음._

## 8. 사고·고장 신고 (긴급 도움)  · 구체성: 중

```
쏘카 앱의 '사고·고장 신고(긴급 도움)' 화면을 디자인해줘. 이용 중 사고나 고장이 났을 때 빠르게 도움받는 화면이야.

구성:
- 상단: "도움이 필요하신가요?" 타이틀 + 안심시키는 한 줄.
- 긴급 액션(가장 강조): 긴급출동/고객센터 전화를 눈에 띄게. 상황의 위급함이 색으로 전달되게.
- 상황 선택: 사고 / 고장 / 기타 문의를 아이콘 리스트(또는 선택 박스)로.
- 주의 안내: "안전한 곳으로 이동한 뒤 신고해 주세요" 같은 Tips/Alert.
- 보조: 자주 묻는 질문, 현재 이용 중인 차량 정보.

상태: 기본 상태.
톤: 침착하고 신뢰감 있게. 급한 상황에서도 헷갈리지 않게, status 색을 적절히.
```

_왜: status 시맨틱 색의 올바른 사용, Alert/Tips, 강조 위계가 분명한 화면. 평소 화면과 톤이 달라 디자인 시스템의 상태 표현을 제대로 검증한다._

## 추가 후보 (한 줄 브리프 — 필요할 때 살 붙여서)

여정을 더 촘촘히 채우거나 분량을 늘리고 싶을 때 쓸 후보들. 위 8개와 컴포넌트가 겹치지 않게 골랐다.

- **예약 일정 선택**: 날짜·시간을 고르는 바텀시트 — DatePicker/TimePicker, 빠른 선택(오늘/내일·1시간 단위) 칩, 하단 '적용'.
- **목적지/쏘카존 검색**: 검색 입력 + 최근 검색 + 추천 목적지 리스트 + 빈 상태.
- **마이페이지**: 프로필 + 멤버십/등급 + 메뉴 리스트(결제수단, 운전면허, 알림설정 등) + 운전면허 인증 상태.
- **쏘카플랜/패스포트 구독 가입**: 혜택 카드 carousel + 요금제 비교(선택 박스) + 하단 '구독 시작'.
- **알림 센터**: 알림 리스트(읽음/안읽음 구분, Badge) + 빈 상태.
- **이벤트/혜택 상세**: 히어로 배너 + 혜택 설명 + 참여 조건 아코디언 + 하단 '참여하기'.
- **부름/편도 신청**: 출발/도착 위치 입력 + 차량 선택 + 요금 안내 + 신청 CTA.
