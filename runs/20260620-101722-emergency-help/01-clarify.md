# Clarify

## Understood request
SOCAR 앱의 '사고·고장 신고(긴급 도움)' 풀스크린입니다. 이용 중 사고나 고장이 난 사용자가 **당황하지 않고 즉시 도움을 받는 것**이 1차 과업이고, 가장 강조되는 1차 액션은 **긴급출동/고객센터 전화 연결**입니다. 모바일 포트레이트, 한국어 카피, 침착·신뢰감 있는 SOCAR 보이스. 기본 상태 1종만 디자인합니다.

## Scope
- In:
  - TopAppBar(뒤로가기/닫기) + 화면 타이틀 「도움이 필요하신가요?」 + 안심시키는 한 줄 부제.
  - **긴급 액션(최상단·최강조):** 긴급출동/고객센터 전화 연결 — 위급함을 status 색으로 전달.
  - **상황 선택:** 사고 / 고장 / 기타 문의 — 아이콘이 있는 SelectionBox(또는 리스트) 형태.
  - **주의 안내:** 「안전한 곳으로 이동한 뒤 신고해 주세요」 — Tips/Alert.
  - **보조 정보:** 자주 묻는 질문(FAQ) 진입 + 현재 이용 중인 차량 정보.
- Out:
  - 전화 연결 후/사고 접수 진행 플로우, 신고 입력 폼, 위치 공유/지도 상세.
  - empty/loading/error 등 추가 상태 (기본 상태 1종만).
  - 실제 전화번호·약관·법적 고지 문구 확정.

## Assumptions (sensible SOCAR / socarframe defaults)
- **풀스크린**으로 디자인 — 진입 빈도/정보량과 "한 화면에서 즉시 행동" 요구상 BottomSheet보다 전체 화면이 적합. (의도가 다르면 plan에서 조정 가능)
- **긴급 액션 = 화면 최상단**(타이틀 바로 아래)에 배치 — 위급 상황에서 한 손·즉시 도달. release checklist의 "한 손 조작/야외 식별"을 만족.
- **위급함 색 = `status-negative`(weak 배경 + regular/strong 포인트)** 로 긴급출동/전화 액션을 표현, **주의 안내 Tips는 `status-caution`** 로 구분. status 색은 액션·경고 영역에만 절제해서 사용(브랜드 `primary`는 보조). — 도메인상 사고=경고/위험 신호가 표준이며 헷갈림을 최소화.
- 긴급 액션은 **전화 연결 1차 + 고객센터 2차** 형태의 강조 카드/버튼으로 구성, 화면 내 강조 액션은 위급 영역 하나로 수렴(One primary action).
- 상황 선택은 **icon 포함 SelectionBox 리스트(사고·고장·기타 문의 3개)**, 각 항목은 탭 시 해당 신고 흐름으로 이동(목적지는 out of scope, 진입점만 표현).
- 현재 이용 중인 차량 정보는 **차량명/번호판 + 간단 식별** 수준의 요약 카드. (실데이터 소스는 plan에서 placeholder)
- FAQ는 **리스트 진입(더보기) 항목** 1~3개 + 전체 보기 진입.
- 카피는 침착·명료한 SOCAR 보이스로 plan 단계에서 확정(이 단계에서 문구 미확정).
- Canvas: 세션 전용 레인, 최상위 프레임 **x≈15000, y≈0** 시작. 프레임명 `[auto] 사고·고장 신고(긴급 도움) — 20260620-101722-emergency-help`.
