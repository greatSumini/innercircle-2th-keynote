# Clarify

## Understood request
쏘카 차량 예약 직전, 사용자가 예약 내용을 마지막으로 확인하고 결제를 완료하는 **전체 화면(full screen)** 이다. 주 사용자는 차량을 예약하려는 쏘카 이용자이고, 단일 primary job은 "예약 내용·요금을 확인하고 결제하여 예약을 완료"하는 것이다. 화면은 상단 바('예약 확인') + 스크롤 본문(예약 요약 / 요금 상세 / 할인 적용 / 결제수단 / 약관 동의) + 하단 고정 영역(최종 결제 금액 + primary CTA)으로 구성된다. 톤은 명료하고 안심되게, 특히 금액 가독성을 강조한다. 기본 상태 하나만 디자인한다.

## Scope
- In:
  - TopAppBar — 타이틀 '예약 확인' (뒤로가기 포함).
  - 예약 요약 카드 — 차량 / 쏘카존 / 이용 일시 / 대여 시간.
  - 요금 상세 — 대여요금 + 보험료 + 예상 주행요금 + 합계(강조), 세부는 Accordion(펼침/접힘).
  - 할인 적용 — 쿠폰 선택(적용된 쿠폰 표시 또는 '쿠폰 적용하기') + 포인트 사용 입력.
  - 결제수단 — 등록된 기본 카드 표시 + '변경' 액션.
  - 약관 동의 — 필수/선택 구분 체크 (Checkbox/CheckboxGroup).
  - 하단 sticky 영역 — 최종 결제 금액 + '결제하고 예약 완료' primary ActionButton.
- Out:
  - 기본 외 상태(로딩 / 에러 / 빈 쿠폰 / 결제 실패 / 결제 완료 후 화면) — 요청이 기본 상태 하나만.
  - 쿠폰 선택 시트, 결제수단 변경 화면 등 연결되는 별도 화면/시트의 내부.
  - 실데이터 연동·금액 계산 로직 (대표 더미 값으로 표현).

## Assumptions (sensible SOCAR / socarframe defaults)
- **모바일 포트레이트, 한국어 카피, socarframe 토큰/컴포넌트 사용** — CLAUDE.md / 도메인 기본값. 명시 불필요.
- **상태는 "기본(default) = 폼이 채워진 결제 가능 상태"** — 쿠폰 1개 적용, 기본 카드 등록됨, 필수 약관 미체크가 아닌 화면 흐름이 보이도록 대표 값을 채운다. CTA는 enabled로 표현(비활성 사유는 별도 상태라 out).
- **요금 Accordion은 '예상 주행요금' 항목에 세부 펼침을 거는 single 모드, 기본 접힘** — 요청의 "세부는 아코디언으로 펼쳐볼 수 있게"를 충족하는 가장 자연스러운 위치. 합계는 항상 노출되어 강조.
- **포인트 사용은 TextField(숫자 입력) + '전액 사용' 보조 액션 한 줄로 표현** — 쏘카 결제 패턴의 표준형. 보유 포인트 잔액을 보조 텍스트로 표시.
- **결제수단은 기본 카드 1장 + 우측 'TextButton/변경'** — 카드사·말미 4자리·아이콘으로 표현.
- **약관은 '전체 동의' 1줄 + 필수 항목 묶음 + 선택 항목 묶음, 각 항목 우측에 보기(>) 가능 표시** — 결제 동의 표준 패턴.
- **금액 가독성: 합계/최종 결제 금액에 가장 강한 타이포 위계(heading) + 텍스트 토큰**을 사용하고, 화면 내 금액은 동일 정렬 규칙(우측 정렬)으로 통일 — 톤 요구("금액이 또렷하게")의 해석. 구체 위계/강조는 plan 단계가 확정.
- **하단 영역은 sticky로 본문 위에 고정, 최종 결제 금액(라벨+값)과 primary CTA를 한 묶음으로** — 요청 명시 + 한 손 조작 원칙.
- **캔버스 배치: top-level 프레임 x≈7800, y≈0** — 요청의 캔버스 레인 지시. 상태가 1개이므로 프레임 1개.
- **카피는 SOCAR 보이스로 plan 단계가 확정** — 정확한 문구·간격·세부 강조는 다음 단계 책임이므로 여기서 묻지 않음.
