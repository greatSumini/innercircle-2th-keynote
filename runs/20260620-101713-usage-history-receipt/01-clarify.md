# Clarify

## Understood request
쏘카 앱의 '이용 내역 & 영수증' 흐름. 첫 번째는 지난 이용을 월별로 묶어 카드 리스트로 보여주는 **이용 내역 리스트** 풀스크린이고, 사용자의 주요 잡(job)은 "내 지난 이용을 훑고 한 건을 골라 결제 내역을 확인"하는 것이다. 두 번째는 카드 탭 시 진입하는 **영수증 상세** 풀스크린으로, 이용 요약 + 항목별 결제 영수증(아코디언으로 세부 펼침) + 결제수단/승인 일시 + 보조 액션을 담는다. 두 화면 모두 모바일 포트레이트, 한국어, socarframe 기준. 톤은 정확하고 신뢰감 있게, **숫자 가독성이 최우선**.

## Scope
- In:
  - 화면 1 — 이용 내역 리스트: TopAppBar + 월별 구분 헤더 + 이용 내역 카드(차량 모델 / 이용 일시 / 쏘카존 / 결제 금액), 카드 탭 → 영수증 상세 진입
  - 화면 1 빈 상태 — 내역이 하나도 없을 때(empty state)
  - 화면 2 — 영수증 상세: 이용 요약(차량·쏘카존·이용 일시·주행 거리) + 결제 영수증(대여/주행/보험/할인 항목별 금액 + 총 결제 금액 강조, 세부는 Accordion 펼침/접힘) + 결제수단·승인 일시 + 보조 액션(영수증 문의 / 같은 차 다시 예약)
- Out:
  - 실제 결제·환불·정정 처리 플로우, 영수증 PDF/이메일 발송 화면
  - "영수증 문의" 진입 후의 상담/CS 화면, "같은 차 다시 예약" 진입 후의 예약 플로우
  - 필터·검색·기간 선택 등 부가 탐색 기능(요청에 없음 — 필요 시 plan에서 보조로만)

## Assumptions (sensible SOCAR / socarframe defaults)
- 두 화면 모두 **풀스크린 프레임**(BottomSheet/모달 아님). 상단은 TopAppBar(리스트: 일반 타이틀 「이용 내역」, 상세: BasicBackButton + 「영수증」). — 리스트→상세가 별도 페이지 전환인 전형적 내역 패턴이고, 영수증은 정보량이 많아 시트보다 풀스크린이 신뢰감/가독성에 유리.
- 영수증 상세는 **상태 1종(정상 결제 완료)** 만 디자인. 부분 환불/결제 실패/정산 중 등 변형 상태는 요청 범위 밖으로 보고 제외. — 요청이 명시한 상태는 리스트/상세/빈 상태 3종.
- 세부 항목 아코디언은 **socarframe Accordion**으로 구성, 기본은 접힘이되 데모 화면에서는 한 항목(예: 주행 또는 대여) 펼친 상태로 보여 구조를 드러냄. `single` 모드, Trigger에 항목명+금액 / Content에 세부 내역. — Accordion 컴포넌트 스펙에 맞는 표준 구성.
- 금액·총 결제 금액은 **숫자 강조 위계**로 처리: 총 결제 금액은 큰 굵은 텍스트(heading 계열) + 항목 금액은 본문, 할인은 음수/색 구분. 통화는 「원」, 1,000 단위 콤마. — "숫자 가독성 최우선" 요구의 socarframe typography 매핑.
- 빈 상태는 일러스트/아이콘 + 안내 문구(예: 「아직 이용 내역이 없어요」) + 1차 행동 유도(예: 「차량 둘러보기」 보조 버튼). — 표준 empty state 패턴.
- 보조 액션 위계: 「영수증 문의」는 **텍스트/링크 버튼**, 「같은 차 다시 예약」은 **보조(secondary) 버튼** 으로 1차 CTA 없이 동등~보조 위계. — 요청의 "텍스트/보조 버튼" 위계를 그대로 따름.
- 리스트 카드는 1차 정보(차량 모델·결제 금액)를 강조하고 일시·쏘카존은 보조 텍스트로 정렬. 카드 단위로 월별 섹션에 그룹화(섹션 헤더 = 「2026년 6월」 식). — Predictability/스캔 용이성.
- 더미 데이터는 한국어 실제 차량/쏘카존 명칭(예: 「아반떼 CN7」, 「캐스퍼」, 「강남역 1번출구 쏘카존」)로 채워 신뢰감 있는 목업 구성. 리스트는 2개월에 걸친 3~4건 정도.
- 캔버스 배치: top-level 프레임을 x=11400, y=0 부근에서 좌→우로 배치(요청 명시). 프레임 명명은 `[auto] 이용 내역 — <run-id> · 리스트`, `· 리스트 빈 상태`, `· 영수증 상세` 식으로 figma-conventions에 맞춤.
