# Clarify

## Understood request
쏘카 앱의 **홈(메인) 화면** — 앱을 켜면 가장 먼저 보이는 풀스크린(모바일 세로) 화면. 주 사용자는 차량 이용을 원하는 일반 고객이고, 화면의 단일 핵심 과업은 **예약 진입(검색 → 쏘카존/목적지)**, 보조 과업은 **진행 중/예정 이용 현황을 한눈에 확인하고 바로 행동(스마트키/예약 상세)** 하는 것이다. 위에서 아래로 스크롤되는 단일 풀스크린이며, **예약 있음 / 예약 없음(첫 화면)** 두 상태를 별도 프레임으로 디자인한다. 톤은 활기차고 깔끔하게, 다음 행동(예약)을 자연스럽게 유도한다.

## Scope
- In:
  - ① 상단 바: 현재 위치/자주 가는 목적지 주소 + 우측 알림 IconButton(읽지 않은 알림 시 dot badge)
  - ② 큰 검색 박스 "어디서 탈까요?" — 쏘카존/목적지 검색 진입점
  - ③ 진행 중/예정 예약 카드: 차량 썸네일 + 모델명 + 이용 시간 + 쏘카존 + 스마트키/예약 상세 버튼 (예약 있음 상태에만)
  - ④ 혜택 배너 carousel: 도트 인디케이터 + 2~3장(신규 할인·주행 리워드 등)
  - ⑤ 퀵메뉴: 아이콘 + 라벨 그리드(쏘카플랜, 부름/편도, 쿠폰함, 이벤트 등)
  - 상태 2종: (A) 예약 있음, (B) 예약 없음(첫 화면)
- Out:
  - 검색 결과/지도/쏘카존 목록 화면 (별도 화면, 진입점만)
  - 알림함, 스마트키, 예약 상세, 쿠폰함 등 퀵메뉴/버튼이 향하는 목적지 화면
  - 하단 글로벌 탭바(BottomNavigation)는 홈 컨테이너 일부지만 별도 명세 없음 → 기본 형태로만 포함(아래 가정 참고)
  - 로딩/에러 상태(요청에 없음)

## Assumptions (sensible SOCAR / socarframe defaults)
- **풀스크린, 모바일 세로 프레임** — 요청에 "앱 진입 시 처음 보이는 화면", 디바이스 별도 명시 없음 → 하네스 기본값(모바일 portrait).
- **상단 바는 TopAppBar 패턴 응용** — Left/Title 슬롯에 위치 주소(아이콘 + 주소 + 펼침 chevron, label 타입 활용), TrailingButtonSlot에 알림 IconButton. dot badge는 미읽음 표시로 노출(예약 있음 버전), 예약 없음 버전은 깔끔히 미노출 또는 노출 중 plan에서 결정.
- **검색 박스는 비입력 진입형(tap → 검색 화면 이동)** — 실제 입력 필드가 아니라 큰 prominent 진입 박스. 플레이스홀더 "어디서 탈까요?", 좌측 검색 아이콘. 강조를 위해 surface/그림자 사용.
- **예약 카드 CTA = ActionButton** — 진행 중(스마트키 가능) 상태는 `fill/primary`(스마트키), 예정 상태는 `outlined/primary` 또는 `fill/secondary`(예약 상세). 한 카드에 CTA 1개. 예약 있음 버전은 예시로 **진행 중 1건**을 기본으로(여러 건은 plan에서 단건/스택 결정).
- **혜택 배너 carousel = Pattern(type="carousel")** — 도트 인디케이터, 3장 기본(신규 할인 / 주행 리워드 / 평일 할인 류), 현재 인덱스 도트 강조.
- **퀵메뉴 = 4-up 아이콘 그리드 1행(+필요 시 "전체" 포함)** — 쏘카플랜, 부름/편도, 쿠폰함, 이벤트 4개 기본. 아이콘 + 라벨, 라벨은 caption 류 타이포.
- **예약 없음(첫 화면) 버전** — ③ 예약 카드 자리를 비우고, 검색 유도를 더 강조(예: "지금 주변 차량을 찾아보세요" 류 진입 강조 또는 빈 자리 축소). empty 처리는 카드 부재 + 검색/혜택 강조로 자연스럽게.
- **하단 글로벌 탭바** — SOCAR 홈은 글로벌 탭(홈/검색/이용내역/마이 류)을 가지므로 화면 하단에 기본 탭바 1개 포함(홈 active). 세부 탭 항목/명세 없음 → 통상 구성으로 배치, plan에서 항목 확정.
- **컬러/타이포/스페이싱/라운드는 socarframe 시맨틱 토큰** — `primary-regular`, `text-secondary`, `spacing-*`, `radius-*`, `heading*/title*` 등. 톤("활기차고 깔끔")은 primary 블루 강조 + 충분한 여백으로 표현. (디자인 시스템이 결정 → 미질의)
- **언어 = 한국어, SOCAR 보이스** (요청 명시).
- **캔버스 배치** — 이 세션 레인: top-level 프레임 x=2400, y=0 부근에서 상태 프레임을 좌→우로 나열, 기존 프레임과 겹치지 않게.
